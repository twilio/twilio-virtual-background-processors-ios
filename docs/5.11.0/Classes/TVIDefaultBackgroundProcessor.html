Overview

To use the TVIDefaultBackgroundProcessor, create an instance of this with either the backgroundImage or the blurFilterRadius and pass that instance as the last parameter (backgroundProcessorDelegate) to the initWithOptions:delegate:backgroundProcessorDelegate: while creating an instance of the TVICameraSource.

TVIDefaultBackgroundProcessor is only available for iOS 17.0 and above iOS versions.

TVIDefaultBackgroundProcessor supports up to 30 fps with video frame resolution of 640X480 on a reasonably powerful device (iPhone 13 or above). Note that higher frame resolution than or higher fps than the recommended above might result in a lower fps and video frames might get dropped.

The TVIDefaultBackgroundProcessor after processing the video frame buffer for virtual background or background blurring, a TVIVideoFrame is passed to the video sink. Only TVIPixelFormatYUV420BiPlanarFullRange pixel format is supported for the imageBuffer property of TVIVideoFrame for the processed frames.

– init

Developers shouldn’t initialize this class without any parameter for either the backgroundImage or the blurFilterRadius.

- (null_unspecified instancetype)init

Discussion

TVIDefaultBackgroundProcessor can not be created with init. To use the default background processor, use one of the two initializers below.

– initWithBackgroundImage:

Initializes a TVIDefaultBackgroundProcessor with a background image.

- (nullable instancetype)initWithBackgroundImage:(nonnull UIImage *)backgroundImage

Parameters

backgroundImage

A UIImage for the virtual background.

Return Value

A TVIDefaultBackgroundProcessor, or nil if one could not be created.

Discussion

Setting an image for this parameter sets the blurFilterRadius property value to 0.0. It is assumed that the initial orientation of the backgroundImage is same as the video frame orientation. So if the backgroundImage orientation is not correct then the virtual background may result in a wrong orientation.

Recommended size for the backgroundImage is 640X480 which is same as the recommended frame size. Having an extremely large image will result in a degraded performance. Also, an image with a different aspect ratio may result in distorted virtual background. Our algorithm only resizes the image to the frame size and no cropping is performed. Having the same aspect ratio for the backgroundImage as the video frame therefore results in a much better user experience.

– initWithBlurFilterRadius:

Initializes a TVIDefaultBackgroundProcessor with gaussian kernel blur radius.

- (nullable instancetype)initWithBlurFilterRadius:(nonnull NSNumber *)blurFilterRadius

Parameters

blurFilterRadius

Gaussian kernel blur radius of type NSNumber initialized with a float value.

Return Value

A TVIDefaultBackgroundProcessor, or nil if one could not be created.

Discussion

When blurFilterRadius is set to 0.0, this parameter will not have any effect in the video frame resulting in an unprocessed frame. Also, setting a value for this parameter sets the backgroundImage property value to nil. This parameter works with any positive float value, our recommendation is a value between 10.0 and 20.0. Any larger number will result in a performance degradation.

  backgroundImage

Background image to be used as the virtual background.

@property (nonatomic, strong, nullable) UIImage *backgroundImage

Discussion

This value can be set during the initialization or it can be changed in runtime. Note that, setting this value to nil will result in unprocessed frames. Also, this property is mutually exclusive to the blurFilterRadius property. So both of these properties can’t be set at the same time. Setting a backgroundImage value sets the blurFilterRadius to 0.0.

It is assumed that the initial orientation of the backgroundImage is same as the video frame orientation. So if the backgroundImage orientation is not correct then the virtual background may result in a wrong orientation. Recommended size for the backgroundImage is 640X480 which is same as the recommended frame size. Having an extremely large image will result in a degraded performance.

Given that there are myriad image formats, sometimes, the virtual background may result in a wrong colorspace. In that case we recommend using a diffent JPEG or PNG image.

  pauseProcessing

Pause virtual background or background blurring.

@property (nonatomic, assign) BOOL pauseProcessing

Discussion

By default, the SDK initializes this property to NO. Setting it to YES pauses the virtual background or background blurring. This property can be toggled while TVIDefaultBackgroundProcessor is active.
